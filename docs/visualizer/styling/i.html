Styling and Appearance

Goals TODO

In INET simulations, multiple visualizers are often used at the same time. INET offers several customization options to make it easier to differentiate visualizations and make the visualizers more recognizable. Several visualizers offers similar settings to make customization easier.

In this showcase, we examine how to customize INET visualizers.

INET version: 3.6
Source files location: inet/showcases/visualization/styling

Line and Arrow

The following example shows how to customize lines and arrows. We create a simulation for this example. The simulation can be run by choosing the Line configuration from the ini file.

The network consists of two AdhocHost type nodes, host1 and host2. In this example, we customize DataLinkVisualizer, but INET has several visualizers that have parameters to customize lines and arrows.

We use the following configuration to customize DataLinkVisualizer.

*.visualizer.*.dataLinkVisualizer.displayLinks = true

*.visualizer.*.dataLinkVisualizer.lineColor = "blue"
*.visualizer.*.dataLinkVisualizer.lineStyle = "dashed"
*.visualizer.*.dataLinkVisualizer.lineWidth = 6
*.visualizer.*.dataLinkVisualizer.lineContactSpacing = 15
*.visualizer.*.dataLinkVisualizer.lineShift = 20

Click image to see the differences between the default and the customized visualization.

A simple color or a list of colors can be set as lineColor. A simple color can be set by using the color's name e.g. "blue" or by using the color's RGB value e.g. "#0000FF". A list of colors can be set by enumerating the colors e.g. "green, yellow, red" or can also be set to a set of "dark" colors or a set of "light" colors. The lineStyle parameter can be set to "solid", "dotted" or "dashed".
The width of the line is represented by a number. The bigger the number, the thicker the line.
If there are multiple lines between the same nodes, we can change the space between the lines by setting the lineShift parameter. The bigger the number, the larger the space between the lines.
We can adjust the space between the end of the of the arrow and the network node by setting the lineContactSpacing parameter. The bigger the number, the larger the space.

Icon

The following example shows how to customize the appearance of icons. A simulation is created for this example, that can be run by choosing the Icon configuration from the ini file.

The simulation consits of two nodes, host1 and host2, are of the type AdhocHost. In this simulation, we use TransportConnectionVisualizer but there are several visualizers in INET that have parameters to customize icon.

We use the following configuration to customize the appearance of transport connection icon.

*.visualizer.*.transportConnectionVisualizer.displayTransportConnections = true

*.visualizer.*.transportConnectionVisualizer.icon = "another_marker"
*.visualizer.*.transportConnectionVisualizer.iconColor = "light"

Click image to see the differences between the default and the customized visualization.

By setting the icon parameter, we can set the path of the icon image which represents transport connection between two network nodes. In this example, we use the another_marker png image from the inet/images/ folder, but you can use your own png image as transport connection icon.
The iconColor parameter can be set as a simple color or as a list of colors. A simple color can be set by using the color's name e.g. "blue" or by using the color's RGB value e.g. "#0000FF". A list of colors can be set by enumerating the colors e.g. "green, yellow, red" or can also be set to a set of "dark" or a set of "light" colors.

At LinkBreakVisualizer and PacketDropVisualizer, icon color can be set by using the iconTintColor parameter instead of the iconColor parameter. These visualizers also have an iconTintAmount parameter. By setting iconTintAmount, the colorization amount of the icon can be set. The parameter's value can be set between zero and one. If we set the parameter to zero, the icon will not be tinted at all. If we set the parameter to one, the icon will be tinted fully.

Font

The following example shows how to customize fonts used in labels. A simulation is created for this example, it can be run by choosing the Font configuration from the ini file.

The network contains two AdhocHost type nodes. We use DataLinkVisualizer for this showcase but there are several visualizers which have parameters to customize fonts.

Labels are enabled by default, they can be disabled by setting the displayLabels parameter to false. To customize fonts used in labels, we set visualizer as follows.

*.visualizer.*.dataLinkVisualizer.displayLinks = true

*.visualizer.*.dataLinkVisualizer.labelFont = "Courier New, 12px, bold"
*.visualizer.*.dataLinkVisualizer.labelColor = "red"

Click the image to see the differences between the default and the customized visualization.

Font family, font size and font style can be set by using the labelFont parameter. You can omit any item from this parameter list, if you do not want to change that attribute. For example, you can set labelFont to "bold" so that the font family and font size remain unchanged. The labelColor parameter can be set to a simple color. It can also be set by using the color's RGB value.

At InterfaceTableVisualizer and StatisticVisualizer, the font parameter is used instead of labelFont, and textColor is used instead of labelColor.

The following example shows how to displace annotations around the nodes. A simulation is created for this example, it can be run by choosing the Annotation configuration from the ini file.

Nodes are type of AdhocHost. In this simulation, we use TransportConnectionVisualizer but there are several visualizers in INET that have annotation displacing parameters.

We use the following configuration to displace annotation.

*.visualizer.*.transportConnectionVisualizer.displayTransportConnections = true

*.visualizer.*.transportConnectionVisualizer.displacementHint = "right"

Click image to see the differences between the default and the customized visualization.

By using the displacementHint parameter, the annotations can be placed in eight directions around the node: topCenter, left, right, bottomCenter, topLeft, topRight, bottomLeft, bottomRight. The parameter has an additional option, it is called any. If any is set, annotations will be placed automatically around the node, as close as possible to the node. We can determine the order of annotation positioning by using the displacementPriority parameter. Zero is the largest priority.
